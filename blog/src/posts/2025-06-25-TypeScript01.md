## 타입스크립트 기초

### 변수에 타입을 정의하는 방법

```jsx
var name: string = "captain";
// 변수 이름 뒤에 :를 붙여 타입 정의
```

### 기본 타입

string number boolean object Array tuple any null undefined

void: 함수의 반환 값이 없을 때 사용

### Tuple이란?

**요소의 개수와 타입이 정확히 지정된 배열**

```tsx
// 1. 튜플 선언: 첫 번째는 string, 두 번째는 number 타입이어야 함
let user: [string, number];

// 2. 올바른 할당
user = ["Alice", 25]; // 성공

// 3. 잘못된 할당 (에러 발생)
user = [25, "Alice"]; // 에러: 순서가 다름
user = ["Bob"]; // 에러: 요소 개수가 부족함
user = ["Charlie", 30, true]; // 에러: 요소 개수가 초과됨
```

React의 useState Hook이 튜플을 반환함

useState는 튜플이기 때문에 무조건 첫번째 요소는 state, 두번째 요소는 setState라고 보장

따라서 `const { state: name, setState: setName } = useState("Alice");` 이런 형태가 아니라

순서만 맞추면 `const [name, setName] = useState("Alice");` 이렇게 사용할 수 있음

**React 내에서의 useState 정의**

```tsx
// S는 개발자가 지정한 상태의 타입 (Generic)
function useState<S>(initialState: S): [S, Dispatch<SetStateAction<S>>];
```

`[S, Dispatch<...>]` 형태의 정확한 튜플로 명시

### 함수의 타입 정의

```tsx
function sayWord(word: string): number {
  return word;
}
```

word라는 매개변수는 string 타입이고, 반환되는 값의 타입은 number임을 명시

### 옵셔널 파라미터

함수에서 받지 않아도 되는 파라미터를 정의할 때 ?를 파라미터 이름 뒤에 붙이면 됨

```tsx
function sayMyName(firstName: string, LastName?: string): string {
  return "my name: " + firstName + " " + lastName;
}
```
